Two different models must reconstruct the same doctrine from the same packets.

## Three Stages

<Steps>
  <Step title="Packet => Claim">
    Parse packet. Normalize aliases (ID: to π:, <- to ←). Extract bundle metadata (^bid, ^cid, ^sec, ^dep). One packet = one claim.
  </Step>
  <Step title="Claim => Section">
    Group by ^sec if present, else by subject root. Order by dependency if topo, else by timestamp.
  </Step>
  <Step title="Section => Doctrine">
    Expand ontology aliases. Apply operation glosses. Strict: one claim = one sentence. Loose: adjacent same-subject may merge.
  </Step>
</Steps>

## Preservation Contract

| Level | Guaranteed | May Vary | Lost |
| --- | --- | --- | --- |
| Packet => Claim | op, confidence, subjects, relations, evidence, numbers, temporal | gloss wording | tokenization, whitespace |
| Claim => Section | membership, IDs, dependency order | section titles | original adjacency |
| Section => Doctrine | ontology, topology, numbers, confidence | paragraphing, tone | rhetoric, typography |

## Fidelity Score

```
F = 100 x (0.35*Ssem + 0.20*Snum + 0.15*Srel + 0.10*Stemp + 0.10*Sonto + 0.10*Sconf)
```

Target: **F >= 95** for doctrine-grade round-tripping. BG-007 Medical achieved **F = 97.6**.
