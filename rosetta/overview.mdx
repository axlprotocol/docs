The Rosetta is the core of AXL Protocol. 75 lines. Any LLM reads it once, becomes fluent. No training. No fine-tuning.

## What It Contains

v3 is a BNF grammar kernel, not a numbered document. It defines:

| Section | Content |
| --- | --- |
| PACKET | Packet shape: `PKT := ID\|OP.CC\|SUBJ\|ARG1\|ARG2\|TEMP [META]` |
| OPERATIONS | 7 cognitive ops with usage rules |
| TAGS | 6 typed subject tags ($, @, #, !, ~, ^) |
| MANIFESTS | Profile, Ontology, Bundle as ordinary packets on the bus |
| LOSS CONTRACT | Required metadata for all bundles (mode, keep, loss, fidelity) |
| DECOMPRESS | 3-step pipeline: Packet to Claim to Sections to Document |
| RULES | 7 rules including backward compatibility with v2.x |
| JSON LOWERING | `application/vnd.axl+json` canonical form via JCS (RFC 8785) |
| GENESIS | Protocol spread tracking via bridge |

## The Design Principle

v3 replaced v2.2 (445 lines of documentation-style spec) with a pure BNF grammar. The grammar is the spec. Any LLM that can read BNF can parse and emit AXL packets without examples or tutorials.

## Fetch It

The live kernel: **[axlprotocol.org/v3](https://axlprotocol.org/v3)**

```bash
curl -s https://axlprotocol.org/v3
```

```python
import urllib.request
rosetta = urllib.request.urlopen("https://axlprotocol.org/v3").read().decode()
```
